13/03/2023

"I got the voice activation to work on my laptop, the biggest problem I faced was an "if" statemet that was preventing the rest of the program to run. The "if" statement was checking if the wake word "dina" was in the command. When the command is recognised and converted and parsed into a string, the wake word was excluded in that sentence when converted into text, removing this if statement resolved the issue. Along with getting the voice assistant working, D.I.N.A can also send json data via MQTT. This will be extremely useful as it means any devices connected to D.I.N.A, can control the house. This makes this method very useful for expanding in the future. Some issues I have faced is that the wake word does not work correctly, meaning you can say any name and it will activate the system. A solution for this is that I dive in to the speech recogniser class and see if I can make it mandatory to have "dina" in the sentence before passing it to the google recogniser. Alternatively, I may be able to train the wake word using porcupine. The final steps to completing D.I.N.A 2.0, is to allow the raspberry pi and arduino to communicate. This will envolve a serial connection between the devices, allowing the house to operate." 13/03/2023 10:22pm
